# ECharts 面试一页纸速查

> 面试前 15 分钟过一遍

## 🔧 核心 API（必须脱口而出）

```js
echarts.init(dom, theme?, opts?)   // 初始化，opts: { renderer: 'canvas'|'svg' }
chart.setOption(option, notMerge?, lazyUpdate?)
chart.resize()                     // 容器大小变化时调用
chart.dispose()                    // 组件卸载必须销毁，防内存泄漏
chart.on('event', fn) / chart.off('event')
chart.showLoading() / chart.hideLoading()
echarts.connect('group') / echarts.disconnect('group')
echarts.registerTheme('name', themeObj)
```

## 📊 Option 结构速记

```
option = {
  // 组件层
  title, tooltip, legend, toolbox, grid, dataZoom, visualMap,

  // 坐标轴
  xAxis: [{ type, data, name }], yAxis: [{ type, name }],

  // 数据层
  dataset: { source: [[...], ...] },
  series: [{ type, data, encode, ... }]
}
```

## 🎯 6 个面试必问题

| # | 问题 | 关键词回答 |
|---|------|-----------|
| 1 | ECharts 初始化做了什么？ | 创建实例、绑定渲染引擎、注册事件系统 |
| 2 | setOption 合并 vs 替换？ | notMerge=false 深度合并 / true 完全覆盖 |
| 3 | 双 Y 轴怎么实现？ | yAxis 数组 + series.yAxisIndex |
| 4 | 组件卸载要注意什么？ | dispose() 销毁实例，否则 canvas 内存泄漏 |
| 5 | 大数据量渲染卡顿？ | sampling: 'lttb' + large:true + progressive 渐进渲染 |
| 6 | dataset vs 传统 data？ | 数据图表分离，Transform 变换，一份数据多图共用 |

## 📐 常用配置值速记

| 场景 | 配置 |
|------|------|
| 堆叠 | `series.stack: 'total'` 同值同组 |
| 环形图 | `pie.radius: ['40%', '70%']` |
| 玫瑰图 | `pie.roseType: 'area'` / `'radius'` |
| 折线变平滑 | `line.smooth: true` |
| 折线面积 | `line.areaStyle: {}` |
| 折线阶梯 | `line.step: 'start'` / `'middle'` / `'end'` |
| 百分比堆叠 | `yAxis.type: 'value'` + `series: [{}, {}]` 用 tooltip formatter 算 |
| tooltip 轴触发 | `tooltip.trigger: 'axis'` |
| tooltip 点触发 | `tooltip.trigger: 'item'` |

## ⚡ 性能优化一图流

```
数据量 < 1000         → 默认 Canvas 即可
数据量 1000~10000     → 开启 large: true
数据量 > 10000        → sampling: 'lttb' 采样
数据量 > 50000        → progressive 渐进渲染 + SVG 渲染器
多图表联动            → echarts.connect('group')
```

## 🔗 事件系统

```
鼠标事件：  click / dblclick / mousedown / mousemove / mouseover / mouseout
组件事件：  legendselectchanged / datazoom / brushselected
全局事件：  finished / rendered
```

## 🧠 加分话术

1. "我习惯用 **dataset** 统一管理数据源，配合 **encode** 做维度映射..."
2. "图表组件我会封装成 **可复用组件**，接收 option 和 events 配置..."
3. "遇到大数据量我会先开 **sampling 采样**，还不够就用 **progressive** 渐进渲染..."
4. "多个图表需要联动时，**echarts.connect** 一把搞定 tooltip 和 dataZoom 同步..."
5. "按需引入：只 import 用到的图表类型和组件，减少打包体积..."

## 🚫 常见坑

- **忘记 dispose** → canvas dom 残留，内存泄漏
- **容器没设宽高** → 图表渲染不出来
- **resize 没做防抖** → 频繁触发 resize 卡死
- **series data 格式不对** → pie 要 [{name, value}]，其他要 []
- **dataset 模式下用了 data** → 两个数据源冲突，谁在前面用谁
